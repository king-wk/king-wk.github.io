运行

JSON

JSON 方法是定义网络拓扑的传统方法。基于AppRunner，该解析器读取 JSON 文件并相应地创建网络

  • p4-src：必须传递给交换机的 p4 程序

  • cli：是否需要在网络启动后激活网络客户端

  • pcap_dump：是否要在交换机接口上激活数据包嗅探，然后将嗅探到的数据包保存在 .pcap 文件中

  • enable_log：启用或禁用开关日志

  • topology：关于拓扑的说明

    • assignment_strategy：使用自动策略为借口分配地址

    • default：应用于每个链接的默认设置的集合。例如强制每个链接的带宽为 10 Mbps

    • links：拓扑中存在的所有链接的列表，还可以为链接指定自定义选项

    • hosts：主机字典，每个主机都有自己的字典来传递选项（例如为 h1 设置默认网关 10.0.0.254）

    • switches：交换机字典（例如为 s1 设置自定义 p4 程序）

运行网络

自动分配策略

默认每个主机仅连接到一个交换机，仅允许交换机和主机，不允许并行链接。

l2

将所有设备置于同一个 IPv4 网络中（10.0.0.0/16）

如果使用主机 h(ID)，交换机 s(ID)，那么 Host IP 分配为 10.0.x.y/16，其中 x 和 y 分别是主机 ID 的高字节和低字节（即其二进制表示）

mixed

混合策略将连接到同一交换机的主机置于同一子网中，将不同的交换机（甚至连在一起的交换机）置于不同的子网中

如果使用主机 h(ID)，交换机 s(ID)，那么 Host IP 分配为 10.x.y.z/24，其中 x 和 y 分别是网关交换机 ID（即其二进制表示）的高字节和低字节，z 是主机 ID；直接连接主机的交换机端口 IP 分配为 10.x.y.254/24，其中 x 和 y 分别是网关交换机 ID（即其二进制表示）的高字节和低字节；连接交换机的交换机端口 IP 分配为 20.sw1.sw2.<1,2>/24，其中 sw1 是第一个交换机的 ID（按照链路定义中的顺序），sw2 是第二个交换机的 ID。最后一个字节如果是 sw1 的接口，则为 1，否则为 2

l3

l3 策略将每台主机置于不同的子网中，该子网与其连接的交换机端口的 IP 地址共享

如果使用主机 h(ID)，交换机 s(ID)，那么 Host IP 分配为 10.x.y.2/24，其中 x 是网关交换机的 ID，y 是主机 ID；直接连接主机的交换机端口 IP 分配为 10.x.y.1/24，其中 x 是网关交换机的 ID，y 是主机 ID；连接交换机的交换机端口 IP 分配为 20.sw1.sw2.<1,2>/24，其中 sw1 是第一个交换机的 ID（按照链路定义中的顺序），sw2 是第二个交换机的 ID。最后一个字节如果是 sw1 的接口，则为 1，否则为 2

拓扑数据库

P4 Utils 有一个内置的拓扑数据库，该数据库在网络启动后自动生成，并保存到执行目录中的 JSON 文件中。可以查询该文件以检索拓扑信息。该框架由 NetworkGraph 实现

任务调度程序

Task Scheduler 允许用户在不同的节点上简单地安排不同的任务（例如流量的生成）。它可以通过网络客户端添加任务，也可以使用文件调度任务。

使用网络客户端安排任务

  • node：节点名称
  • exe：运行的可执行文件
  • argX：传递函数位置参数
  • start：相对于当前时间的任务延迟秒数
  • duration：以秒为单位的任务持续时间（如果持续时间小于或等于 0，则任务没有时间限制）
  • keyX 和 kwargX：传递函数的关键字参数

例如，h1 ping h2 10 秒
